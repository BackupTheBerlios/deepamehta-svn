Upgrading an existing installation

With version ...TODO..., the file system layout of DeepaMehta has changed. If you're upgrading from an older version, please do not simply install the new version into the existing directories. Instead, perform the following steps to upgrade your installation:

  1. Install DeepaMehta into a new directory (e. g. /home/fflintstone/DeepaMehta).
  2. Take a look at the file install/client/dms.rc of your current installation. You will find a (single) line that looks like this:
    default 7557 RelationalCorporateMemory org.gjt.mm.mysql.Driver jdbc:mysql://127.0.0.1/ DeepaMehta?user=dmuser&password=sssecrettt&useUnicode=true&characterEncoding=latin1
    In the following steps, replace all the colored passages with the corresponding values from your file.
  3. Manually create a new directory inside the data directory (e. g. data/default).
  4. Copy the following directories including all files from your current installation to the directory you just created. You will find the directories in install/client of your current installation.
    • backgrounds
    • documents
    • icons
    • images
    • sounds
    • stylesheets
  5. Create a text file named instances.xml in your new DeepaMehta installation directory and add the following lines:
    <?xml version="1.0" encoding="ISO-8859-1"?>
    <dmi:instances xmlns:dmi="http://www.deepamehta.de/schema/InstanceDefinition.xsd" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="http://www.deepamehta.de/schema/InstanceDefinition.xsd InstanceDefinition.xsd">
    <instance description="any description text" id="default">
    <monolithic>
    <cm class="de.deepamehta.service.RelationalCorporateMemory">
    <property name="user" value="dmuser"/>
    <property name="password" value="sssecrettt"/>
    <property name="host" value="127.0.0.1"/>
    <property name="database" value="DeepaMehta"/>
    <property name="driver" value="org.gjt.mm.mysql.Driver"/>
    </cm>
    </monolithic>
    </instance>
    </dmi:instances>

You should be able to start your instance from the launch pad now.
